# 3. 系统总体架构文档

## 3.1 系统概述

OptiBus 是面向海南黎安试验区的需求响应式动态摆渡车调度系统，采用 B/S 架构，包含乘客端、司机端、管理员端三个前端视图，以及基于 FastAPI 的后端服务和 SQLite/Redis 数据层。

## 3.3 技术选型

| 层级   | 技术                        | 说明                 |
| ------ | --------------------------- | -------------------- |
| 前端   | Vue3 + Vite + Pinia         | 响应式 SPA，三端视图 |
| 后端   | FastAPI + Uvicorn           | 异步 Python Web 框架 |
| 数据库 | SQLite                      | 轻量级关系型存储     |
| 缓存   | Redis                       | 实时位置与客流缓存   |
| 通信   | HTTP REST + WebSocket       | 请求响应 + 实时推送  |

## 3.4 核心调度算法

调度引擎基于三个核心步骤：

1. **瞬时客流热力计算**：按站点聚合在站候车人数，生成热力分布
2. **最近车辆筛选**：基于 GPS 坐标计算各车辆到热力站点的距离，筛选可用车辆
3. **比例分配**：按客流比例动态分配车辆，实现线路间柔性互帮
